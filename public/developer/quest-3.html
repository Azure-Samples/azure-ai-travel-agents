File: src/api/src/mcp/mcp-http-client.ts

This file defines the MCPClient class, which implements an HTTP-based client for connecting to MCP servers. It handles tool discovery, invocation, and notifications, providing a robust interface for interacting with tools hosted on MCP servers.

Highlights

  • connect: Establishes a connection to the MCP server using the HTTP transport.
  • listTools: Retrieves the list of tools available on the connected MCP server.
  • callTool: Invokes a specific tool with the provided arguments and returns the result.

Code

async connect() {
  await this.client.connect(this.transport);
  console.log('Connected to server');
}
  • The connect method initializes the HTTP transport and establishes a connection to the MCP server.
  • It ensures the client is ready to send requests and receive responses from the server.
  • This method is critical for enabling communication between the orchestrator and the MCP server.
async listTools() {
  const result = await this.client.listTools();
  return result.tools;
}
  • The listTools method fetches the available tools from the MCP server.
  • This dynamic discovery mechanism allows the system to adapt to changes in the toolset without requiring redeployment.
  • It ensures that the orchestrator always has an up-to-date view of the server's capabilities.
async callTool(name: string, toolArgs: string) {
  console.log(`Calling tool ${name} with arguments:`, toolArgs);

  return await this.client.callTool({
    name,
    arguments: JSON.parse(toolArgs),
  });
}
  • The callTool method sends a request to execute a specific tool with the given arguments and returns the result.
  • It uses defensive programming by parsing the tool arguments to ensure they are in the correct format.
  • This method is central to enabling task execution across the distributed agent network.

File: src/api/src/mcp/mcp-sse-client.ts

This file provides an alternative implementation of the MCP client using Server-Sent Events (SSE) for streaming data. While marked as a legacy implementation, it demonstrates how the system supports multiple transport protocols.

Highlights

  • connect: Establishes an SSE connection to the MCP server.
  • listTools: Retrieves the list of available tools via the SSE transport.
  • callTool: Invokes a tool and processes the result using SSE.

Code

connect() {
  return tracer.startActiveSpan("connect", async (span) => {
    log("Connecting to MCP SSE server");
    try {
      await this.client.connect(this.transport);
      log("Connected to MCP SSE server");
      span.end();
      return this.client;
    } catch (error: any) {
      log("Error connecting to MCP SSE server:", error);
      span.setStatus({ code: 2, message: (error as Error).message });
      span.end();
      throw new Error(
        `Failed to connect to MCP SSE server: ${(error as Error).message}`
      );
    }
  });
}
  • The connect method establishes a connection to the MCP server using SSE.
  • It uses OpenTelemetry for tracing, providing visibility into the connection lifecycle.
  • Error handling ensures that connection failures are logged and propagated appropriately.
async listTools() {
  return tracer.startActiveSpan("listTools", async (span) => {
    log("Tools", this.tools);
    const toolsResult = await this.client.listTools();
    this.tools = toolsResult.tools;
    log("Tools: ", toolsResult);
    span.end();
    return toolsResult;
  });
}
  • The listTools method retrieves the list of tools available on the SSE-connected MCP server.
  • OpenTelemetry tracing is used to monitor the tool discovery process.
  • This method ensures that the orchestrator can dynamically adapt to changes in the toolset.
async callTool(toolName: string, args: Record<string, any>) {
  console.log(`Called ${toolName} with params:`, args);
  return tracer.startActiveSpan("processQuery", async (span) => {
    log("Tools", this.tools);

    const toolResult = await this.client.callTool({
      name: toolName,
      arguments: args,
    });

    log("Tool result", toolResult);
    span.end();
    return toolResult;
  });
}
  • The callTool method invokes a specific tool and returns the result.
  • It uses structured logging and tracing to monitor tool execution.
  • This method demonstrates how the system handles tool invocation in a streaming context.

File: src/api/src/mcp/mcp-tools.ts

This file provides utility functions for managing MCP clients and tools. It includes logic for initializing clients, discovering tools, and handling connectivity issues.

Highlights

  • client: Factory function for creating MCP clients based on the specified transport type.
  • mcpToolsList: Discovers tools across multiple MCP servers and handles unreachable servers gracefully.
  • McpServerDefinition: Defines the configuration schema for MCP servers.

Code

function client(config: LlamaIndexMCPClientOptions): MCPSSEClient | MCPHTTPClient {
  console.log(`Initializing MCP client`);
  console.log(`Using configuration:`, {config});

  if (config.type === "sse") {
    // legacy implementation using SSE
    return new MCPSSEClient("llamaindex-sse-client", config.url, config.accessToken);
  } else {
    return new MCPHTTPClient("llamaindex-http-client", config.url, config.accessToken);
  }
}
  • The client function creates an MCP client based on the specified transport type (HTTP or SSE).
  • This factory pattern enables flexibility in supporting different communication protocols.
  • The function ensures consistent initialization of clients with the required configuration.
export async function mcpToolsList(config: McpServerDefinition[]) {
  return await Promise.all(
    config.map(async ({ id, name, config }) => {
      const { url, type } = config;
      const mcpClient = client(config);
      
      try {
        console.log(`Connecting to MCP server ${name} at ${url}`);
        await mcpClient.connect();
        console.log(`MCP server ${name} is reachable`);
        const tools = await mcpClient.listTools();

        console.log(`MCP server ${name} has ${tools.length} tools`);
        return {
          id,
          name,
          url,
          type,
          reachable: true,
          selected: id !== "echo-ping",
          tools,
        };
      } catch (error: unknown) {
        console.error(
          `MCP server ${name} is not reachable`,
          (error as Error).message
        );
        return {
          id,
          name,
          url,
          type,
          reachable: false,
          selected: false,
          tools: [],
          error: (error as Error).message,
        };
      }
    })
  );
}
  • The mcpToolsList function discovers tools across multiple MCP servers.
  • It handles connectivity issues by marking unreachable servers and logging errors.
  • This function ensures that the system remains operational even when some servers are unavailable.
