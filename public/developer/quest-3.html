File: src/api/src/mcp/mcp-sse-client.ts

This file defines an SSE-based MCP client, providing a legacy implementation for connecting to MCP servers. The MCPClient class handles tool discovery, invocation, and connection management using Server-Sent Events (SSE) as the transport protocol.

Highlights

  • connect establishes an SSE-based connection to the MCP server.
  • listTools fetches the tools available on the server and updates the internal tool registry.
  • callTool invokes a specific tool, logging the process for observability.

Code

connect() {
  return tracer.startActiveSpan("connect", async (span) => {
    log("Connecting to MCP SSE server");
    try {
      await this.client.connect(this.transport);
      log("Connected to MCP SSE server");
      span.end();
      return this.client;
    } catch (error: any) {
      log("Error connecting to MCP SSE server:", error);
      span.setStatus({ code: 2, message: (error as Error).message });
      span.end();
      throw new Error(
        `Failed to connect to MCP SSE server: ${(error as Error).message}`
      );
    }
  });
}
  • Establishes a connection to the MCP server using the SSE transport.
  • Uses OpenTelemetry spans to trace the connection process for monitoring and debugging.
  • Implements robust error handling to manage connectivity issues gracefully.
  • Logs connection status and errors for better observability.
async listTools() {
  return tracer.startActiveSpan("listTools", async (span) => {
    log("Tools", this.tools);
    const toolsResult = await this.client.listTools();
    this.tools = toolsResult.tools;
    log("Tools: ", toolsResult);
    span.end();
    return toolsResult;
  });
}
  • Fetches the list of tools from the MCP server and updates the internal tool registry.
  • Uses OpenTelemetry spans to trace the tool discovery process.
  • Logs the fetched tools for debugging and monitoring purposes.
  • Simplifies integration with the orchestrator by providing a consistent tool discovery interface.
async callTool(toolName: string, args: Record<string, any>) {
  console.log(`Called ${toolName} with params:`, args);
  return tracer.startActiveSpan("processQuery", async (span) => {
    log("Tools", this.tools);

    const toolResult = await this.client.callTool({
      name: toolName,
      arguments: args,
    });

    log("Tool result", toolResult);
    span.end();
    return toolResult;
  });
}
  • Invokes a specific tool on the MCP server with the provided arguments.
  • Uses OpenTelemetry spans to trace the tool invocation process.
  • Logs the invocation and results for better observability and debugging.
  • Demonstrates the flexibility of the SSE-based implementation for tool execution.

File: src/api/src/mcp/mcp-tools.ts

This file combines the HTTP and SSE MCP clients, providing a unified interface for managing MCP servers and their tools. The client function dynamically initializes the appropriate client based on the server configuration.

Highlights

  • client dynamically selects the appropriate MCP client (HTTP or SSE) based on the configuration.
  • mcpToolsList connects to multiple MCP servers and fetches their tool lists.
  • McpServerDefinition defines the configuration schema for MCP servers.

Code

function client(config: LlamaIndexMCPClientOptions): MCPSSEClient | MCPHTTPClient {
  console.log(`Initializing MCP client`);
  console.log(`Using configuration:`, {config});

  if (config.type === "sse") {
    // legacy implementation using SSE
    return new MCPSSEClient("llamaindex-sse-client", config.url, config.accessToken);
  } else {
    return new MCPHTTPClient("llamaindex-http-client", config.url, config.accessToken);
  }
}
  • Dynamically selects the appropriate MCP client based on the server configuration.
  • Supports both HTTP and SSE protocols, ensuring backward compatibility.
  • Logs the client initialization process for debugging and monitoring.
  • Demonstrates the extensibility of the client design.
export async function mcpToolsList(config: McpServerDefinition[]) {
  return await Promise.all(
    config.map(async ({ id, name, config }) => {
      const { url, type } = config;
      const mcpClient = client(config);
      
      try {
        console.log(`Connecting to MCP server ${name} at ${url}`);
        await mcpClient.connect();
        console.log(`MCP server ${name} is reachable`);
        const tools = await mcpClient.listTools();

        console.log(`MCP server ${name} has ${tools.length} tools`);
        return {
          id,
          name,
          url,
          type,
          reachable: true,
          selected: id !== "echo-ping",
          tools,
        };
      } catch (error: unknown) {
        console.error(
          `MCP server ${name} is not reachable`,
          (error as Error).message
        );
        return {
          id,
          name,
          url,
          type,
          reachable: false,
          selected: false,
          tools: [],
          error: (error as Error).message,
        };
      }
    })
  );
}
  • Connects to multiple MCP servers and fetches their tool lists.
  • Handles connectivity issues gracefully, logging errors for unreachable servers.
  • Returns a structured list of tools, enabling dynamic registration in the orchestrator.
  • Demonstrates the scalability of the MCP client design for multi-server environments.
