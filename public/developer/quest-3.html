This file contains a legacy implementation of the MCP client using Server-Sent Events (SSE). While it provides similar functionality to the HTTP-based client, it demonstrates a different transport mechanism and includes OpenTelemetry tracing for observability.

  • connect: Initializes the SSE transport and establishes a connection to the MCP server.
  • listTools: Retrieves the list of tools from the MCP server and caches them locally.
  • callTool: Executes a tool on the MCP server and returns the result.

This file orchestrates the interaction between the MCP clients and the broader system. It includes logic for selecting the appropriate client type (HTTP or SSE) and managing tool discovery across multiple MCP servers.

src/api/src/mcp/mcp-http-client.ts

async connect() {
  await this.client.connect(this.transport);
  console.log('Connected to server');
}

async listTools() {
  const result = await this.client.listTools();
  return result.tools;
}

async callTool(name: string, toolArgs: string) {
  console.log(`Calling tool ${name} with arguments:`, toolArgs);

  return await this.client.callTool({
    name,
    arguments: JSON.parse(toolArgs),
  });
}
  • Establishes a connection to the MCP server using HTTP transport.
  • Lists all tools available on the server, returning their metadata.
  • Invokes a specific tool by name with JSON-parsed arguments, ensuring flexibility in input structure.
  • Demonstrates clear separation of concerns for connection, discovery, and execution.
  • Logging provides transparency into operations for debugging and monitoring.

src/api/src/mcp/mcp-sse-client.ts

connect() {
  return tracer.startActiveSpan("connect", async (span) => {
    log("Connecting to MCP SSE server");
    try {
      await this.client.connect(this.transport);
      log("Connected to MCP SSE server");
      span.end();
      return this.client;
    } catch (error: any) {
      log("Error connecting to MCP SSE server:", error);
      span.setStatus({ code: 2, message: (error as Error).message });
      span.end();
      throw new Error(
        `Failed to connect to MCP SSE server: ${(error as Error).message}`
      );
    }
  });
}

async listTools() {
  return tracer.startActiveSpan("listTools", async (span) => {
    log("Tools", this.tools);
    const toolsResult = await this.client.listTools();
    this.tools = toolsResult.tools;
    log("Tools: ", toolsResult);
    span.end();
    return toolsResult;
  });
}
  • Uses OpenTelemetry spans for tracing connection and tool listing operations.
  • Provides robust error handling during connection initialization, ensuring clear error messages.
  • Caches tools locally for efficient reuse and reduced server load.
  • Tracing integration enables performance monitoring and debugging across distributed systems.
  • Highlights the legacy nature of SSE transport compared to the newer HTTP-based approach.

src/api/src/mcp/mcp-tools.ts

function client(config: LlamaIndexMCPClientOptions): MCPSSEClient | MCPHTTPClient {
  console.log(`Initializing MCP client`);
  console.log(`Using configuration:`, {config});

  if (config.type === "sse") {
    // legacy implementation using SSE
    return new MCPSSEClient("llamaindex-sse-client", config.url, config.accessToken);
  } else {
    return new MCPHTTPClient("llamaindex-http-client", config.url, config.accessToken);
  }
}

export async function mcpToolsList(config: McpServerDefinition[]) {
  return await Promise.all(
    config.map(async ({ id, name, config }) => {
      const { url, type } = config;
      const mcpClient = client(config);
      
      try {
        console.log(`Connecting to MCP server ${name} at ${url}`);
        await mcpClient.connect();
        console.log(`MCP server ${name} is reachable`);
        const tools = await mcpClient.listTools();

        console.log(`MCP server ${name} has ${tools.length} tools`);
        return {
          id,
          name,
          url,
          type,
          reachable: true,
          selected: id !== "echo-ping",
          tools,
        };
      } catch (error: unknown) {
        console.error(
          `MCP server ${name} is not reachable`,
          (error as Error).message
        );
        return {
          id,
          name,
          url,
          type,
          reachable: false,
          selected: false,
          tools: [],
          error: (error as Error).message,
        };
      }
    })
  );
}
  • The client function abstracts transport selection, ensuring flexibility for different MCP server implementations.
  • mcpToolsList iterates through server configurations, connecting to each and retrieving tool metadata.
  • Gracefully handles unreachable servers by logging errors and returning fallback data.
  • Demonstrates the use of asynchronous operations for concurrent server interactions.
  • Provides detailed logging for operational visibility and debugging.
