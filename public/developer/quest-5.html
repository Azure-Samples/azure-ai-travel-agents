src/ui/src/app/chat-conversation/chat-conversation.service.ts: Chat Service Logic

The ChatService is the backbone of the Streaming Chat UI. It manages the user and agent message streams, integrates with the ApiService for backend communication, and processes real-time events. This service ensures that messages are synchronized and that the UI updates dynamically as new data arrives.

Highlights

  • fetchAvailableTools retrieves the list of tools from the backend and filters them based on user selection.
  • sendMessage handles user message submission, updates the message buffer, and initiates the streaming process.
  • processAgentEvents processes real-time events from the backend, updating the message state and handling specific agent events like AgentToolCallResult.
  • updateAndNotifyAgentChatMessageState updates the chat message state buffer and notifies subscribers of changes.
  • showErrorMessage displays error notifications when issues occur during communication.

async fetchAvailableTools() {
  const toolsResult = await this.apiService.fetchAvailableTools();
  if (toolsResult) {
    this.tools.set(toolsResult.tools.filter((tool) => tool.selected));
  }
}
  • This method retrieves the list of available tools from the backend via the ApiService.
  • Filters tools based on the selected property, ensuring only user-approved tools are displayed.
  • Demonstrates how the service maintains a reactive state using Angular's signal.
async sendMessage(event: Event) {
  if ((event as KeyboardEvent).shiftKey) {
    return;
  }

  const messageText = this.userMessage();
  if (!messageText.trim()) return;

  this.messagesBuffer.push({
    role: 'user',
    content: messageText,
    reasoning: [],
    timestamp: new Date(),
  });

  this.userMessage.set('');
  this.isLoading.set(true);
  this.assistantMessageInProgress.set(false);

  await this.apiService.streamChatMessage(
    messageText,
    this.tools().filter((tool) => tool.selected)
  );
}
  • Captures user input, validates it, and pushes it to the message buffer for display.
  • Invokes the streamChatMessage method of ApiService to start the streaming process.
  • Uses Angular signals to manage loading states and message progress indicators.
private processAgentEvents(event?: ChatEvent) {
  if (event && event.type === 'metadata') {
    this.currentAgentName.set(event.data?.agentName || null);
    this.agentEventsBuffer.push(event);

    const delta: string = event.data?.delta || '';

    switch (event.event) {
      case 'StartEvent':
        this.updateAndNotifyAgentChatMessageState(delta, {
          metadata: {
            events: this.agentEventsBuffer,
          },
        });
        this.assistantMessageInProgress.set(false);
        break;
      case 'AgentToolCallResult':
        let state = {};
        if (typeof event.data.raw === 'string') {
          state = {
            reasoning: [
              {
                content: event.data.raw,
              },
            ],
          };
        }
        this.updateAndNotifyAgentChatMessageState(delta, state);
        break;
    }
  }
}
  • Processes real-time events received from the backend, such as metadata updates and tool call results.
  • Updates the chat message state with new data and reasoning content.
  • Demonstrates the use of event-driven programming for dynamic UI updates.

src/ui/src/app/services/api.service.ts: Backend Communication

The ApiService acts as the bridge between the frontend and backend, enabling the Streaming Chat UI to fetch tools and stream chat messages. It leverages server-sent events (SSE) for real-time updates.

Highlights

  • fetchAvailableTools retrieves the list of tools from the backend API.
  • streamChatMessage manages the streaming of user messages and processes server responses in real-time.
  • handleApiError provides consistent error handling for API requests, ensuring robust communication.

Code

async fetchAvailableTools(): Promise<{tools: Tools[]} | void> {
  try {
    const response = await fetch(`${this.apiUrl}/api/tools`, {
      method: 'GET',
      headers: {
        'Content-Type': 'application/json',
      },
    });
    if (!response.ok) {
      const { error } = await response.json();
      return this.handleApiError(
        new Error(error || 'An error occurred while fetching tools'),
        response.status
      );
    }
    return (await response.json());
  } catch (error) {
    return this.handleApiError(error, 0);
  }
}
  • Sends a GET request to the backend to fetch the list of available tools.
  • Implements error handling for network issues and server-side errors.
  • Demonstrates the use of modern fetch API with async/await patterns.
async streamChatMessage(message: string, tools: Tools[]) {
  try {
    const response = await fetch(`${this.apiUrl}/api/chat`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ message, tools }),
    });

    if (!response.ok) {
      const { error } = await response.json();
      return this.handleApiError(
        new Error(error || 'An error occurred'),
        response.status
      );
    }

    const decoder = new TextDecoder('utf-8');

    if (!response.body) {
      throw new Error('Readable stream not supported');
    }

    this.chatStreamState.next({ type: 'START' });

    for await (const chunk of response.body) {
      const value = decoder.decode(chunk, { stream: true });

      const jsonValues = value.trim().split(/\n\n+/);

      for (const jsonValue of jsonValues) {
        try {
          const parsedData = JSON.parse(jsonValue);
          this.chatStreamState.next({
            type: 'MESSAGE',
            event: parsedData,
          });
        } catch (error) {
          console.error('Error parsing JSON chunk:', error);
        }
      }
    }

    this.chatStreamState.next({ type: 'END' });
  } catch (error) {
    this.handleApiError(error, 0);
  }
}
  • Streams user messages and processes server responses in real-time using server-sent events.
  • Handles chunked responses by decoding and parsing JSON data incrementally.
  • Demonstrates robust stream handling and error management.
