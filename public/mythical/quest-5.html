The ChatService is the core spellbook that manages the flow of messages between the user and the agents. It orchestrates the chat state, processes agent responses, and handles errors. This service ensures a seamless conversation experience by leveraging Angular's reactive tools like BehaviorSubject and signals.

Highlights

  • fetchAvailableTools: Dynamically retrieves a list of tools from the MCP server and updates the state with selected tools.
  • sendMessage: Sends user messages to the agents and initiates the streaming process for agent responses.
  • processAgentEvents: Processes various types of events from the agents, including metadata, tool calls, and output streams.
  • updateAndNotifyAgentChatMessageState: Updates the state of the chat messages and notifies subscribers of changes.
  • showErrorMessage: Displays user-friendly error messages when issues occur during the chat process.

async fetchAvailableTools() {
  const toolsResult = await this.apiService.fetchAvailableTools();
  if (toolsResult) {
    this.tools.set(toolsResult.tools.filter((tool) => tool.selected));
  }
}
  • Dynamically fetches available tools from the MCP server.
  • Filters tools to include only the ones marked as selected.
  • Updates the tools signal, which is reactive and automatically triggers UI updates.
async sendMessage(event: Event) {
  if ((event as KeyboardEvent).shiftKey) {
    return;
  }

  const messageText = this.userMessage();
  if (!messageText.trim()) return;

  this.messagesBuffer.push({
    role: 'user',
    content: messageText,
    reasoning: [],
    timestamp: new Date(),
  });

  this.userMessage.set('');
  this.isLoading.set(true);
  this.assistantMessageInProgress.set(false);

  await this.apiService.streamChatMessage(
    messageText,
    this.tools().filter((tool) => tool.selected)
  );
}
  • Sends the user's message to the agents.
  • Adds the message to the messagesBuffer for immediate UI feedback.
  • Initiates the streaming process for agent responses using selected tools.
  • Manages loading state to provide visual feedback to the user.
private processAgentEvents(event?: ChatEvent) {
  if (event && event.type === 'metadata') {
    this.currentAgentName.set(event.data?.agentName || null);
    this.agentEventsBuffer.push(event);

    const delta: string = event.data?.delta || '';

    switch (event.event) {
      case 'StartEvent':
        this.updateAndNotifyAgentChatMessageState(delta, {
          metadata: {
            events: this.agentEventsBuffer,
          },
        });
        this.assistantMessageInProgress.set(false);
        break;
      case 'StopEvent':
        this.updateAndNotifyAgentChatMessageState(delta, {
          metadata: {
            events: this.agentEventsBuffer,
          },
        });
        this.assistantMessageInProgress.set(false);
        this.isLoading.set(false);
        break;
      case 'AgentToolCallResult':
        let state = {};
        if (typeof event.data.raw === 'string') {
          state = {
            reasoning: [
              {
                content: event.data.raw,
              },
            ],
          };
        }
        this.updateAndNotifyAgentChatMessageState(delta, state);
        break;
      // Other event cases omitted for brevity
    }
  }
}
  • Processes streaming events from the MCP server.
  • Handles different types of agent events, such as StartEvent, StopEvent, and AgentToolCallResult.
  • Updates the chat state with new data and notifies subscribers for real-time UI updates.

The ApiService acts as the bridge between the Crystal Chat Interface and the agent tools. It handles API calls, streams agent responses, and manages errors. This service is crucial for enabling real-time communication with MCP tools.

Highlights

  • fetchAvailableTools: Fetches the list of tools from the MCP server and returns it for use in the UI.
  • streamChatMessage: Streams chat messages to the MCP tools and processes their responses in real-time.
  • handleApiError: Handles errors during API calls and updates the chat stream state accordingly.

Code

async fetchAvailableTools(): Promise<{tools: Tools[]} | void> {
  try {
    const response = await fetch(`${this.apiUrl}/api/tools`, {
      method: 'GET',
      headers: {
        'Content-Type': 'application/json',
      },
    });
    if (!response.ok) {
      const { error } = await response.json();
      return this.handleApiError(
        new Error(error || 'An error occurred while fetching tools'),
        response.status
      );
    }
    return (await response.json());
  } catch (error) {
    return this.handleApiError(error, 0);
  }
}
  • Fetches tools from the MCP server using a GET request.
  • Implements error handling to provide meaningful feedback in case of failures.
  • Returns the list of tools for integration into the chat interface.
async streamChatMessage(message: string, tools: Tools[]) {
  try {
    const response = await fetch(`${this.apiUrl}/api/chat`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ message, tools }),
    });

    if (!response.ok) {
      const { error } = await response.json();
      return this.handleApiError(
        new Error(error || 'An error occurred'),
        response.status
      );
    }

    const decoder = new TextDecoder('utf-8');

    if (!response.body) {
      throw new Error('Readable stream not supported');
    }

    this.chatStreamState.next({ type: 'START' });

    for await (const chunk of response.body) {
      const value = decoder.decode(chunk, { stream: true });

      const jsonValues = value.trim().split(/\n\n+/);

      for (const jsonValue of jsonValues) {
        try {
          const parsedData = JSON.parse(jsonValue);
          this.chatStreamState.next({
            type: 'MESSAGE',
            event: parsedData,
          });
        } catch (error) {
          console.error('Error parsing JSON chunk:', error);
        }
      }
    }

    this.chatStreamState.next({ type: 'END' });
  } catch (error) {
    this.handleApiError(error, 0);
  }
}
  • Streams user messages to the MCP server and processes the response as a real-time stream.
  • Uses TextDecoder to parse streaming chunks and extract JSON data.
  • Updates the chatStreamState to reflect the progress of the streaming operation.
