src/api/src/mcp/mcp-http-client.ts: The HTTP Conjurer

This file defines the MCPClient class, which uses the HTTP protocol to connect to MCP servers. It facilitates tool discovery and invocation while emitting magical notifications when tools are updated. The class leverages a streamable HTTP transport to maintain efficient communication with the servers.

Highlights

  • connect: Establishes a connection to the MCP server and ensures readiness for tool operations.
  • listTools: Retrieves a list of available tools from the server, enabling dynamic tool discovery.
  • callTool: Invokes a specific tool on the server with provided arguments, executing its magical purpose.

Code

src/api/src/mcp/mcp-http-client.ts

async connect() {
  await this.client.connect(this.transport);
  console.log('Connected to server');
}
  • Establishes a connection to the MCP server using the transport layer.
  • Ensures the client is ready to send and receive tool-related requests.
  • Logs the connection status for debugging and monitoring.
async listTools() {
  const result = await this.client.listTools();
  return result.tools;
}
  • Fetches the list of tools available on the connected MCP server.
  • Uses the listTools method of the MCP client for dynamic discovery.
  • Returns an array of tool metadata, enabling the orchestration of magical workflows.
async callTool(name: string, toolArgs: string) {
  console.log(`Calling tool ${name} with arguments:`, toolArgs);

  return await this.client.callTool({
    name,
    arguments: JSON.parse(toolArgs),
  });
}
  • Invokes a specific tool by its name, passing the required arguments.
  • Parses the arguments from a JSON string to ensure compatibility with the MCP protocol.
  • Logs the invocation details for transparency and debugging.
