src/api/src/orchestrator/llamaindex/providers/index.ts: Provider Strategy Coordination

This file serves as the central hub for selecting and initializing AI model providers based on environment configurations. It dynamically routes requests to specific provider implementations, ensuring flexibility and scalability. The llm() function is the cornerstone, orchestrating model selection and error handling to adapt to diverse deployment scenarios.

Highlights

  • llm(): A factory function that selects the appropriate AI model provider by reading the LLM_PROVIDER environment variable and invoking the corresponding implementation.
  • LLMProvider: A type definition that lists valid provider options, ensuring clarity and consistency in model selection.
  • Error handling: Ensures that invalid or missing LLM_PROVIDER values trigger descriptive errors, preventing misconfigurations.

Code

src/api/src/orchestrator/llamaindex/providers/index.ts

import dotenv from "dotenv";
dotenv.config();

import { llm as azureOpenAI } from "./azure-openai.js";
import { llm as foundryLocal } from "./foundry-local.js";
import { llm as githubModels } from "./github-models.js";
import { llm as dockerModels } from "./docker-models.js";
import { llm as ollamaModels } from "./ollama-models.js";

type LLMProvider =
  | "azure-openai"
  | "github-models"
  | "foundry-local"
  | "docker-models"
  | "ollama-models";

const provider = (process.env.LLM_PROVIDER || "") as LLMProvider;

export const llm = async () => {
  switch (provider) {
    case "azure-openai":
      return azureOpenAI();
    case "github-models":
      return githubModels();
    case "docker-models":
      return dockerModels();
    case "ollama-models":
      return ollamaModels();
    case "foundry-local":
      return foundryLocal();
    default:
      throw new Error(
        `Unknown LLM_PROVIDER "${provider}". Valid options are: azure-openai, github-models, foundry-local, docker-models, ollama-models.`
      );
  }
};
  • The llm() function dynamically selects the AI model provider based on the LLM_PROVIDER environment variable, ensuring adaptability across deployment environments.
  • The LLMProvider type defines valid options, promoting clarity and reducing the risk of invalid configurations.
  • Error handling is robust, providing descriptive feedback when the LLM_PROVIDER value is missing or unsupported, ensuring system stability.

src/api/src/orchestrator/llamaindex/providers/azure-openai.ts: Azure OpenAI Integration

This file implements the Azure OpenAI model provider, supporting authentication via API keys or Azure Managed Identity. It adapts to both local and production environments, ensuring secure and seamless access to AI models.

Highlights

  • llm(): Configures the Azure OpenAI provider using environment variables for endpoint, deployment, and authentication.
  • DefaultAzureCredential and ManagedIdentityCredential: Enable authentication flexibility, supporting both local and production scenarios.
  • getBearerTokenProvider(): Facilitates secure token generation for Azure Cognitive Services, ensuring reliable access to resources.

Code

src/api/src/orchestrator/llamaindex/providers/azure-openai.ts

import { openai } from "llamaindex";
import {
  DefaultAzureCredential,
  getBearerTokenProvider,
  ManagedIdentityCredential,
} from "@azure/identity";

const AZURE_COGNITIVE_SERVICES_SCOPE =
  "https://cognitiveservices.azure.com/.default";

export const llm = async () => {
  console.log("Using Azure OpenAI");

  const isRunningInLocalDocker = process.env.IS_LOCAL_DOCKER_ENV === "true";
  
  if (isRunningInLocalDocker) {
    console.log(
      "Running in local Docker environment, Azure Managed Identity is not supported. Authenticating with apiKey."
    );
    
    return openai({
      azure: {
        endpoint: process.env.AZURE_OPENAI_ENDPOINT,
        deployment: process.env.AZURE_OPENAI_DEPLOYMENT,
        apiKey: process.env.AZURE_OPENAI_API_KEY,
      },
    });
  }
  
  let credential: any = new DefaultAzureCredential();
  const clientId = process.env.AZURE_CLIENT_ID;
  if (clientId) {
    console.log("Using Azure Client ID:", clientId);
    credential = new ManagedIdentityCredential({
      clientId,
    });
  }

  const azureADTokenProvider = getBearerTokenProvider(
    credential,
    AZURE_COGNITIVE_SERVICES_SCOPE
  );

  return openai({
    azure: {
      azureADTokenProvider,
      endpoint: process.env.AZURE_OPENAI_ENDPOINT,
      deployment: process.env.AZURE_OPENAI_DEPLOYMENT,
    },
  });
};
  • The llm() function adapts to local and production environments, using API keys for local setups and Azure Managed Identity for production deployments.
  • DefaultAzureCredential and ManagedIdentityCredential provide flexible authentication mechanisms, supporting diverse deployment scenarios.
  • getBearerTokenProvider() ensures secure and reliable token generation for accessing Azure Cognitive Services.

import { openai } from "llamaindex";

export const llm = async () => {
  console.log("Using GitHub Models");
  return openai({
    baseURL: "https://models.inference.ai.azure.com",
    apiKey: process.env.GITHUB_TOKEN,
    model: process.env.GITHUB_MODEL,
  });
};
  • The llm() function configures the GitHub Models provider with environment variables for API keys and model selection, ensuring flexibility.
  • Model selection is dynamic, allowing easy switching between AI models based on deployment needs.
