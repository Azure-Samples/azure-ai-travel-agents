File: src/api/src/mcp/mcp-sse-client.ts

The mcp-sse-client.ts file implements an alternative client for connecting to MCP servers using Server-Sent Events (SSE). This legacy implementation demonstrates how streaming protocols can be used for tool discovery and invocation.

Highlights

  • connect: Establishes an SSE-based connection to the MCP server.
  • listTools: Retrieves available tools and caches them locally.
  • callTool: Invokes a tool and logs its execution details.
  • cleanup: Gracefully closes the SSE connection.

Code

connect() {
  return tracer.startActiveSpan("connect", async (span) => {
    log("Connecting to MCP SSE server");
    try {
      await this.client.connect(this.transport);
      log("Connected to MCP SSE server");
      span.end();
      return this.client;
    } catch (error: any) {
      log("Error connecting to MCP SSE server:", error);
      span.setStatus({ code: 2, message: (error as Error).message });
      span.end();
      throw new Error(
        `Failed to connect to MCP SSE server: ${(error as Error).message}`
      );
    }
  });
}
  • Uses OpenTelemetry tracing to monitor the connection lifecycle.
  • Handles connection errors gracefully, logging relevant details.
  • Ensures the span is ended properly, even in failure scenarios.
async listTools() {
  return tracer.startActiveSpan("listTools", async (span) => {
    log("Tools", this.tools);
    const toolsResult = await this.client.listTools();
    this.tools = toolsResult.tools;
    log("Tools: ", toolsResult);
    span.end();
    return toolsResult;
  });
}
  • Retrieves and logs the list of tools from the server.
  • Caches the tools locally for quick access.
  • Uses tracing to monitor the listTools operation.
async cleanup() {
  await this.transport.close();
}
  • Closes the SSE connection to release resources.
  • Ensures a clean shutdown of the client.

File: src/api/src/mcp/mcp-tools.ts

The mcp-tools.ts file provides utilities for managing multiple MCP clients. It dynamically initializes clients based on configuration and retrieves tool information from all connected servers.

Highlights

  • client: Factory function that initializes an MCP client based on the specified protocol (http or sse).
  • mcpToolsList: Connects to multiple MCP servers and retrieves their tools.
  • McpServerDefinition: Type definition for server configurations.

Code

function client(config: LlamaIndexMCPClientOptions): MCPSSEClient | MCPHTTPClient {
  console.log(`Initializing MCP client`);
  console.log(`Using configuration:`, {config});

  if (config.type === "sse") {
    // legacy implementation using SSE
    return new MCPSSEClient("llamaindex-sse-client", config.url, config.accessToken);
  } else {
    return new MCPHTTPClient("llamaindex-http-client", config.url, config.accessToken);
  }
}
  • Dynamically selects the appropriate client type based on configuration.
  • Supports both HTTP and SSE protocols for backward compatibility.
  • Logs configuration details for debugging.
export async function mcpToolsList(config: McpServerDefinition[]) {
  return await Promise.all(
    config.map(async ({ id, name, config }) => {
      const { url, type } = config;
      const mcpClient = client(config);
      
      try {
        console.log(`Connecting to MCP server ${name} at ${url}`);
        await mcpClient.connect();
        console.log(`MCP server ${name} is reachable`);
        const tools = await mcpClient.listTools();

        console.log(`MCP server ${name} has ${tools.length} tools`);
        return {
          id,
          name,
          url,
          type,
          reachable: true,
          selected: id !== "echo-ping",
          tools,
        };
      } catch (error: unknown) {
        console.error(
          `MCP server ${name} is not reachable`,
          (error as Error).message
        );
        return {
          id,
          name,
          url,
          type,
          reachable: false,
          selected: false,
          tools: [],
          error: (error as Error).message,
        };
      }
    })
  );
}
  • Connects to multiple MCP servers in parallel.
  • Handles connectivity errors gracefully and logs them.
  • Returns detailed information about each server, including its tools.
